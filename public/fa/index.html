راهنمای DNS

راهنمای جامع DNS و انواع رکوردهای دی‌ان‌اس

در این بخش با مفهوم DNS، روش تست DNS آنلاین و کاربرد مهم‌ترین رکوردهای دامنه آشنا می‌شوید.

DNS یا دی‌ان‌اس چیست؟

DNS یا سیستم نام دامنه، یکی از اصلی‌ترین بخش‌های اینترنت است. وقتی کاربر آدرس یک سایت مثل example.com را وارد می‌کند، DNS کمک می‌کند آن نام دامنه به اطلاعات فنی قابل استفاده برای سرورها تبدیل شود؛ مثل آدرس IP، رکوردهای ایمیل، Name Serverها و تنظیمات امنیتی دامنه.

به زبان ساده، DNS مانند دفترچه راهنمای اینترنت عمل می‌کند. کاربر نام دامنه را وارد می‌کند و DNS مشخص می‌کند این دامنه باید به کدام سرور، IP یا سرویس متصل شود.

اگر DNS دامنه درست تنظیم نشده باشد، ممکن است سایت باز نشود، ایمیل‌ها دریافت نشوند، SSL به مشکل بخورد یا رکوردهای جدید با تأخیر در اینترنت دیده شوند.

تست DNS آنلاین NetworkMaster چطور کار می‌کند؟

وقتی یک دامنه را در NetworkMaster بررسی می‌کنید، نتیجه فقط از یک نقطه نمایش داده نمی‌شود. درخواست DNS از نود اختصاصی ما بررسی می‌شود و در کنار آن با یک Resolver خارجی معتبر مقایسه می‌گردد.

این مقایسه کمک می‌کند بفهمید رکوردهای DNS واقعاً اعمال شده‌اند یا هنوز بعضی Resolverها مقدار قبلی را از Cache برمی‌گردانند.

چه زمانی به تست DNS آنلاین نیاز داریم؟

  • بعد از انتقال سایت به سرور یا هاست جدید
  • بعد از تغییر Name Serverهای دامنه
  • بعد از تنظیم رکوردهای ایمیل مثل MX و TXT
  • بعد از تغییر رکوردهای A، AAAA، CNAME یا TXT
  • وقتی سایت برای بعضی کاربران باز می‌شود و برای بعضی دیگر نه

آشنایی با انواع رکوردهای DNS

رکورد A

دامنه یا زیردامنه را به یک آدرس IPv4 متصل می‌کند و معمولاً برای اتصال سایت به سرور وب استفاده می‌شود.

رکورد AAAA

مانند رکورد A است، اما برای آدرس‌های IPv6 استفاده می‌شود.

رکورد NS

مشخص می‌کند کدام Name Serverها مسئول مدیریت رکوردهای DNS دامنه هستند.

رکورد CNAME

یک دامنه یا زیردامنه را به نام دیگری متصل می‌کند؛ مثل اتصال www یا app به مقصد دیگر.

رکورد MX

مشخص می‌کند ایمیل‌های دامنه باید به کدام Mail Server ارسال شوند.

رکورد TXT

برای ذخیره اطلاعات متنی، تأیید مالکیت دامنه، SPF، DKIM، DMARC و تنظیمات امنیت ایمیل استفاده می‌شود.

رکورد SOA

اطلاعات مدیریتی DNS Zone مثل Name Server اصلی و مقادیر زمانی به‌روزرسانی را نگه می‌دارد.

رکورد CAA

مشخص می‌کند کدام صادرکننده‌های گواهی SSL اجازه دارند برای دامنه شما Certificate صادر کنند.

رکورد PTR

برای Reverse DNS استفاده می‌شود و یک IP را به نام دامنه یا Hostname برمی‌گرداند.

رکورد SRV

محل یک سرویس خاص را همراه با پورت، اولویت و وزن مشخص می‌کند.

رکورد DS

بخشی از DNSSEC است و برای ایجاد زنجیره اعتماد بین Zone والد و فرزند استفاده می‌شود.

رکورد DNSKEY

کلیدهای عمومی DNSSEC را نگه می‌دارد و برای اعتبارسنجی پاسخ‌های امضاشده DNS استفاده می‌شود.

چرا نتیجه تست DNS در نقاط مختلف متفاوت است؟

بعد از تغییر رکوردهای DNS، نتیجه همیشه در همه جا هم‌زمان به‌روزرسانی نمی‌شود. دلیل اصلی این موضوع Cache شدن رکوردها در Resolverها و سرویس‌دهنده‌های مختلف اینترنتی است.

هر رکورد DNS مقداری به نام TTL دارد. TTL مشخص می‌کند یک Resolver تا چه مدت می‌تواند مقدار یک رکورد را در Cache نگه دارد.
