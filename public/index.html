DNS Guide

How DNS checking works

DNS is the system that translates a domain name into the technical records browsers, mail servers, and other services need to find your website. When you check a domain here, NetworkMaster sends the query to our own DNS node and also compares it with a trusted external resolver, so you can see how the domain is resolving from different points of view.

This is useful when you have changed a DNS record, moved a website to a new server, configured email records, or want to confirm that a domain is returning the expected values.

A record

Points a domain to an IPv4 address. This is one of the most common records used for connecting a domain to a web server.

AAAA record

Works like an A record, but for IPv6 addresses. It is used when a service supports modern IPv6 connectivity.

NS record

Shows which name servers are responsible for managing the DNS zone of the domain.

CNAME record

Creates an alias from one hostname to another. It is commonly used for subdomains such as www or app.

MX record

Defines which mail servers receive email for the domain. Priority values decide which mail server should be tried first.

TXT record

Stores text-based information. TXT records are often used for SPF, DKIM, domain verification, and security-related settings.

SOA record

Contains administrative information about the DNS zone, including the primary name server and timing values.

CAA record

Controls which certificate authorities are allowed to issue SSL certificates for the domain.

PTR record

Used for reverse DNS lookups. It maps an IP address back to a hostname.

SRV record

Defines the location of specific services, including port and priority information.

DS record

Used in DNSSEC to connect a child zone to its parent zone and help validate DNS trust.

DNSKEY record

Stores public keys used by DNSSEC to verify signed DNS responses.

Why results may be different

DNS records can take time to update across different resolvers. If one location shows the new value and another still shows the old value, it usually means the DNS change is still propagating or cached somewhere along the path.
